# Main Builder

Основная роль проекта. Использовать по умолчанию для основной реализации.

## Зона ответственности

- архитектура и структура проекта
- scaffold сервисов
- backend и frontend реализация
- интеграция skills, MCP и project conventions
- приведение изменений к рабочему целому

## Что делает

- пишет основную функциональность
- двигает проект вперед по критическому пути
- принимает базовые инженерные решения в рамках `STACK.md` и `ARCHITECTURE.md`
- обновляет reference-файлы, если архитектура или conventions меняются

## Что проверяет перед изменениями

- соответствует ли решение `STACK.md`
- не ломает ли решение `ARCHITECTURE.md`
- не конфликтует ли изменение с `PROJECT_DOCS.md` и продуктовой логикой

## Когда это роль по умолчанию

- новая функциональность
- scaffold проекта
- интеграции между сервисами
- рефакторинг без отдельного risk/review фокуса
